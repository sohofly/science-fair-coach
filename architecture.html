科展小助手：現行系統架構

依據目前程式碼整理｜靜態前端、Supabase 後端、OpenAI 選用服務｜2026-07-20

架構型態

無獨立傳統伺服器；前端部署於 GitHub Pages，業務 API、驗證、資料庫與檔案全部由 Supabase 承擔。

狀態設計

本機 localStorage 負責即時畫面狀態；登入班級後，重要事件、題目、研究架構與實驗紀錄同步到後端。

權限邊界

教師使用 Supabase Auth＋RLS；學生使用 Edge Function 驗證的自訂 token。service role 與 OpenAI 金鑰只留在函式環境。

主要風險

app.js 與 portal.js 已偏大型單檔；本機與雲端雙狀態可能不同步；附件目前交給 AI 的只有文字與檔名，並未解析檔案內容。
